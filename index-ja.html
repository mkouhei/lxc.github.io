LXC - Linux コンテナ

Linux カーネルの持つコンテナ機能のためのユーザスペースツール

LXC について

LXC は Linux カーネルが持つコンテナ機能のためのユーザスペースのインターフェースです．
Linux ユーザがシステムコンテナやアプリケーションコンテナを簡単に作成したり管理したりするためのパワフルな API とシンプルなツールを提供しています．

機能

最新の LXC では，プロセスを隔離するために以下のカーネルの機能を使います:

  • 名前空間 (ipc, uts, mount, pid, network and user)
  • Apparmor, SELinux プロファイル
  • Seccomp ポリシー
  • Chroot (pivot_root を使います)
  • ケーパビリティ
  • Control groups (cgroups)

    • LXC はよく強力な chroot と本格的な仮想マシンの中間のようなもののように見なされます． LXC の目的は，可能な限り標準的な Linux のインストール環境に近く，しかし別々のカーネルは必要ないような環境を作ることです．

    コンポーネント

    LXC は，現時点ではいくつかのコンポーネントから構成されています．

    • liblxc ライブラリ
    • いくつかの言語に対するバインディング (python3, lua, Go)
    • コンテナを操作するための標準的なツール群
    • コンテナ作成のためのテンプレート

    ライセンス

    LXC は，コードのほとんどの部分が GNU LGPLv2.1+ の条項に基づいてリリースされています．Android 互換である部分は二条項BSDライセンス (2-clause BSD license) に基づいており，GNU GPLv2 ライセンスに基づいてリリースされているバイナリやテンプレートもあります．

    使い方

    詳細な使い方は LXC に含まれる man ページでご覧になれます．基本的な作成，開始，停止方法は以下のようになります: 

    lxc-create -t ubuntu -n p1
lxc-start -n p1 -d
lxc-ls --fancy p1
lxc-stop -n p1

    この例は "p1" という名前の Ubuntu Linux のコンテナを作成し，バックグラウンドでそれを起動しています．そして，そのコンテナの IP アドレスなどの情報を表示させて，最後にそのコンテナを停止しています．

    以上の例と同等の事を python3 API を使って行うと以下のようになります: 

    import lxc
container = lxc.Container("p1")
container.create("ubuntu")
container.start()
container.get_ips()
container.stop()

    Contribute するには

    開発に関する議論はほとんど メーリングリスト 上で行われています．
    通常，master ブランチにマージしたり，レビューしてもらうためには，このメーリングリストにメールを送ります．
    Alpha リリースや RC リリースはタグ付けされます．パッケージ化されるディストリビューションもあります．

    Ubuntu ユーザは開発版のパッケージを ppa:ubuntu-lxc/daily から取得して利用することも可能です．

    コミュニティについて

    先に書いたように，議論はほとんどメーリングリスト上で行われています．
    メーリングリストは 2 つ存在します:

    • lxc-devel: パッチを送付したり開発に関する議論を行う
    • lxc-users: ユーザが扱う話題について

    #lxcontainers on irc.freenode.net で議論されることもあります．

    stable リリースについて

    LXC の stable リリースは現時点では 2013 年 4 月にリリースされた 0.9.0 です．

    次の stable リリースは LXC 1.0 になる予定です．2014 年の 2 月初旬に予定されています．LXC 1.0 は LXC としては stable ブランチとしてメンテナンスされる最初のリリースとなる予定です．

    つまり，誰もパッチを送ってくる事がなくなるまで，LXC 1.0 のバグフィックスのためのリリースを行います．LXC 1.0 は 5 年間はセキュリティ対応やバグフィックスの行われる Ubuntu 14.04 LTS に含まれます．この作業は直接 upstream 版に対して行われ，Ubuntu は標準でバグフィックスのリリースを行います．
